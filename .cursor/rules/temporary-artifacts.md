# Temporary Artifacts and Gitignore Rules

## Never Commit These

### Generated per-run directories
- `**/npu_validation/e2e_triton_to/` -- regenerated every e2e run by `generate_testcase.py`
- `**/npu_validation/<testcase>/` -- regenerated by PTOAS sample runs (e.g., `Abs/npu_validation/abs/`)
- `**/build/` directories under sample paths

### Binary artifacts
- `*.bin` files under `npu_validation/` -- test inputs/outputs
- `*_cpu.bin` files -- CPU-sim output
- `golden_*.bin`, `numpy_golden_*.bin` -- golden reference files
- Compiled binaries (`cpu_sim_test`, `*_sim`, etc.)

### Docker-generated files
- Anything written to PTOAS samples by Docker containers is ephemeral

## Already in .gitignore

The repo's `.gitignore` covers build artifacts, Python caches, editor files, and the `.cursor/` directory itself.

## /tmp Investigation Files

When running manual experiments, place temporary scripts and build artifacts under `/tmp/`. Clean them up after investigation. Known patterns from past work:

- `/tmp/ptoas_e2e_*.sh` -- one-off test scripts
- `/tmp/ptoas_build*` -- intermediate build outputs
- `/tmp/gen_host_stub.py` -- host stub generation script
- `/tmp/official_build/` -- comparison builds of official Ascend samples
